Change Font Color | Scheme | L4D2 Modding

File scheme adalah file resource list yang mengatur banyak kendali seperti, warna, fonts, borders, control dan lainnya. tidak semua kendali ada pada skema terkadang sudah di override oleh script atau melalui file resource kendali element UI itu sendiri, maksud override disini kamu ingin merubah warna nama pada teammate yang ada diatas kepala mereka, walaupun sudah kamu rubah lewat scheme atau cara paksa lain ini tidak akan bisa karena sudah di override atau warna pada nama teammate sudah di tangani oleh script atau file resource

sekarang mari kita lihat beberapa file scheme dan buat apa saja, Kita bahas satu-satu:

1. clientscheme.res

file ini mengambil kendali pada tampilan UI In-game, seperti scoreboard, font HUD, warna pada HUD, dan lainnya, intinya berkaitan di ingame saja dan file ini dipanggil dan digunakan hanya di In-game saja.

2. l4d360uischeme.res

file ini sama saja dengan diatas, hanya saja l4d360uischeme mengambil kendali pada tampilan UI diluar game seperti Mainmenu, video setting window, audio setting, warna, fonts dan lainnya, intinya berkaitan di luar game saja.

3. chatscheme.res

file ini juga sama saja dengan diatas, dan file ini hanya mengambil kendali pada Chat UI saja, kamu biasanya melihat ketika chat message di in-game dan kamu melihat ada tulisan, warna, scrollbar, window, dan lain-lain dan file ini dipanggil dan digunakan hanya di Chat UI saja.

INFO: ingat tidak semua kendali ada pada skema terkadang sudah di override oleh script atau file resource

Getting Started

Agar pemahaman lebih mudah langsung kita praktekan pada kasus sederhana dengan buat mod, mengubah warna font pada HUD

Siapkan tool-tool ini, saya rekomendasi:

1. GCFScape

2. VPKedit

3. Visual Studio Code or Notepad whatever your Text editor

Change Font Color by Scheme File

Kita akan mengganti warna font melalui file scheme, berikut langkah-langkah:

  • Pertama kita siapkan folder baru beri nama "my_mod"
  • Buka my_mod dan buat folder baru beri nama "resource", di dalam sebuah mod, folder dengan nama resource berisikan file-file resource list

  • Selanjutnya kita perlu menyalin file clientscheme.res pada left 4 dead 2, sekarang pergi ke folder:
    "YOUR_STEAM_FOLDER\steamapps\common\Left 4 Dead 2\left4dead2"
    cari file pak01_dir.vpk dan buka dengan GCFScape

  • lanjut buka folder resource, di sana ada banyak file cukup cari saja file clientscheme.res
  • Jika sudah ditemukan Tahan click pada file clientscheme.res drag ke folder: "my_mod\resource" yang telah dibuat tadi,

  • Selanjutnya, kita susun ke VPK dalam satu folder "my_mod", tapi sebelum itu kita perlu membuat satu file plain-text di posisi paling luar didalam folder "my_mod" buat file kosong beri nama addoninfo.txt

    "YOUR_STEAM_FOLDER\steamapps\common\Left 4 Dead 2\left4dead2\addons",
    lalu click tombol Save

Jika sebelumnya kita mengganti warna font melalui file scheme, sekarang kita akan mengganti warna font dengan langsung ke control UI file resource list nya, Kita akan membuat custom Hitungan Health baru dan memberinya warna disana, berikut langkah-langkah:

  • Pertama kembali ke folder "my_mod" buka folder resource buat folder baru, beri nama "ui", lalu buka folder "ui", buat lagi folder baru beri nama "hud"
  • Selanjutnya pergi kembali ke folder:
    "YOUR_STEAM_FOLDER\steamapps\common\Left 4 Dead 2\left4dead2"
    Buka kembali file pak01_dir.vpk dengan GCFScape
  • Kita hanya memerlukan file localplayerpanel.res, file ini menangani resource UI pada HUD Local Player kita sendiri mulai dari Healthbar, HealthNumber, Name, Avatar, dan lainnya bisa dicari sendiri, sebenarnya ada satu file lagi berkaitan yaitu localplayerpanel_incap.res, file ini akan terpanggil ketika player dalam kondisi Incapitated, tapi untuk sekarang kita buat sederhana saja

    lokasi file ini ada di dalam pak01_dir.vpk: "resource\ui\hud"

    Beberapa penjelasan properti beserta value pada resource list diatas:
    • "NewHealthNumber": adalah nama pada elemen UI.
    • ControlName: adalah kendali untuk elemen UI seperti Label, Button, Panel, EditablePanel, Frame, lain-lain.
    • fieldName: adalah nama pada elemen UI.
    • xpos: adalah posisi dalam hitungan Horizontal, simpelnya dari awalan x yang berarti horizontal, dan pos singkatan dari position
    • ypos: adalah posisi dalam hitungan Vertical, simpelnya dari awalan y yang berarti vertical, dan pos singkatan dari position
    • zpos: adalah posisi dalam depan dan belakang, simpelnya ini seperti anda mengedit video atau photo dan ada bermacam-macam layer, jadi semakin tinggi angka semakin berada paling depan elemen-elemen UI lainnya, semakin kecil atau minus semakin berada pada belakang elemen-element UI lainnya.
    • wide: adalah panjang dari elemen tersebut
    • tall: adalah lebar dari elemen tersebut
    • visible: type valuenya adalah boolean 0 adalah hide, 1 adalah terlihat pada layar
    • enabled: type valuenya adalah boolean 0 adalah mati atau tidak berfungsi, 1 adalah aktif atau berfungsi
    • labeltext: type value string juga bisa numeric, contoh diatas labeltext diisi dengan "%HealthNumber%", tanda % adalah untuk menggunakan variable dengan value yang dinamis dan variable HealthNumber hanya bisa diakses di dalam file localplayerpanel.res dan localplayerpanel_incap.res, properti `labeltext` ini hanya ada pada kendali seperti Label, Button, RichText, dan terkait lainnya.
    • textAlignment: adalah posisi align pada text, properti ini hanya ada pada kendali seperti Label, Button, RichText, dan terkait lainnya, beberapa value bisa digunakan:
      "west" = left
      "east" = right
      "north" = top
      "south" = bottom
      "north-west" = top left
      "south-west" = bottom left
      "north-east" = top right
      "south-east" = bottom right
    • font: penentuan seperti apa font digunakan untuk elemen itu sendiri, contoh diatas dengan nama HUDHealth, kendali font ini ada pada file scheme, dengan kasus diatas adalah file localplayerpanel.res file ini berkaitan dengan clientscheme.res untuk mengatur kendali scheme pada file, kamu bisa menemukannya dan mengeditnya, bahkan kamu juga bisa buat kendali font baru disana.
    • fgcolor_override: properti untuk memberi warna pada text.
    • bgcolor_override: properti untuk memberi warna pada background.
    • defaultFgcolor_override: sama seperti diatas, properti untuk memberi warna default pada text, biasanya saya hanya menggunakannya untuk kendali seperti Label dengan memaksa elemen, berbarengan dengan menambahkan ini.
    • defaultBgcolor_override: properti untuk memberi warna default pada background.

Custom Scheme Color Then Implement To Specific Resource List

Jika sebelumnya kita mengganti warna font spesifik dari control UI file resource list nya, sekarang kita akan mengganti warna font pada satu control UI file resource list tapi mengambil warna custom dari file Scheme seperti clientscheme.res, Kita akan membuat elemen HealthNumber lagi tapi dengan custom warna skema yang di implementasikan ke elemen UI, berikut langkah-langkah:

  • Kembali ke Text Editor, buka localplayerpanel.res lalu buat elemen UI baru di bawah elemen `NewHealthNumber` seperti dibawah ini:

    my_mod > resource > ui > hud > localplayerpanel.res

    // Other Code ... "NewHealthNumber" // New HealthNumber with clientscheme.res { "ControlName" "Label" "fieldName" "NewHealthNumber" "xpos" "86" "ypos" "32" "wide" "70" "tall" "26" "visible" "1" "enabled" "1" "labelText" "%HealthNumber%" "textAlignment" "west" "font" "HUDHealth" "zpos" "2" "fgcolor_override" "0 255 255 255" "defaultFgcolor_override" "0 255 255 255" // add this line if above not working } "NewHealthNumber_WithClientScheme" // New HealthNumber with Cyan text color { "ControlName" "Label" "fieldName" "NewHealthNumber_WithClientScheme" "xpos" "116" "ypos" "32" "wide" "70" "tall" "26" "visible" "1" "enabled" "1" "labelText" "%HealthNumber%" "textAlignment" "west" "font" "HUDHealth" "zpos" "2" "fgcolor_override" "MyColorScheme" } }

    Sperti yang kau lihat, disana pada properti fgcolor_override dengan value MyColorScheme ini akan mencari warna skema ke file clientscheme.res dengan key warna MyColorScheme, saat ini kita belum ada key warna tersebut mari kita buat itu sekarang.
  • Selanjutnya pergi ke file clientscheme.res kita akan buat warna dengan key sama dengan MyColorScheme, nama, besar, kecil huruf key warna harus sama dengan yang di localplayerpanel.res, buat di atas di dalam scheme Colors, seperti dibawah ini:

    my_mod > resource > clientscheme.res

    // // TRACKER SCHEME RESOURCE FILE // // sections: // colors - all the colors used by the scheme // basesettings - contains settings for app to use to draw controls // fonts - list of all the fonts used by app // borders - description of all the borders // // Scheme { //Name - currently overriden in code //{ // "Name" "ClientScheme" //} //////////////////////// COLORS /////////////////////////// Colors { // My Custom Colors "MyColorScheme" "190 0 255 255" // base colors "Orange" "255 176 0 255" "OrangeDim" "255 176 0 120" "LightOrange" "188 112 0 128"

    Jika sudah simpan.
  • Setelah itu seperti biasa buka VPKedit, kita susun folder "my_mod" dan simpan di folder:
    "YOUR_STEAM_FOLDER\steamapps\common\Left 4 Dead 2\left4dead2\addons"
  • Buka game Left 4 Dead 2 lihat perubahannya, kira-kira seperti dibawah ini:

Tadi kita sudah membuat mod sederhana untuk merubah warna pada suatu elemen UI pada font, mulai dari yang pertama kita sudah mengganti warna dengan cara merubah dari file scheme pada clientscheme.res, juga kita telah membuat HealthNumber baru dengan nama `NewHealthNumber` dan memberi warna langsung ke elemennya menggunakan properti fgcolor_override, dan terakhir kita membuat HealthNumber lagi tapi dengan properti fgcolor_override mengambil warna ke file clientscheme.res
