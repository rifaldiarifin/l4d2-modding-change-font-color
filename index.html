Change Font Color | Scheme | L4D2 Modding

The scheme file is a resource list file that manages many controls such as, colors, fonts, borders, and others. not all controls are in the scheme sometimes it has been overridden by the script or through the UI element control resource file itself, the meaning of override here, with an example you want to change the color of the name on the teammate that is above their head, even though you have changed it through the scheme or other forced methods this will not be possible because it has been overridden or the color on the teammate's name has been handled by the script or resource file.

Now let's take a look at some scheme files and what they are for, let's go through them one by one:

1. clientscheme.res

This file takes control of the In-game UI, such as the scoreboard, HUD, colors on the HUD, etc., basically related to ingame only and this file is called and used only in In-game.

2. l4d360uischeme.res

This file is the same as above, it's just that l4d360uischeme takes control of the UI display outside the game such as Mainmenu, video setting window, audio settings, colors, fonts and others, basically related outside the game only.

3. chatscheme.res

This file is also the same as above, and this file only takes control of the Chat UI, you usually see when chat messages are in-game and you see text, colors, scrollbars, windows, etc. and this file is called and used only in the Chat UI.

INFO: Remember not all controls are in the schema, sometimes they are overridden by scripts or resource files.

For easier understanding, let's practice it on a simple case by creating a mod, changing the font color on the HUD, let's do it.

Getting Started

Set up and install these tools, I recommend:

1. GCFScape

2. VPKedit

3. Visual Studio Code or Notepad whatever your Text editor

  • We will add a new HealthNumber, type the code below in the row after the `HealthNumber` is up to you to determine the name or fieldname, for now I name it `NewHealthNumber` as below:

    my_mod > resource > clientscheme.res

    // Other code "HealthNumber" { "ControlName" "Label" "fieldName" "HealthNumber" "xpos" "56" [!$OSX] "xpos" "60" [$OSX] "ypos" "32" "wide" "70" "tall" "26" "visible" "1" "enabled" "1" "labelText" "%HealthNumber%" "textAlignment" "west" "font" "HUDHealth" "zpos" "2" } "NewHealthNumber" // New HealthNumber with Cyan text color { "ControlName" "Label" "fieldName" "NewHealthNumber" "xpos" "86" // position will be right side of HealthNumber above "ypos" "32" "wide" "70" "tall" "26" "visible" "1" "enabled" "1" "labelText" "%HealthNumber%" "textAlignment" "west" "font" "HUDHealth" "zpos" "2" "fgcolor_override "0 255 255 255" "defaultFgcolor_override "0 255 255 255" // add this line if above not working } }

    Explanation:
    • "NewHealthNumber": is the name of the UI element.
    • ControlName: are controls for UI elements such as Label, Button, Panel, EditablePanel, Frame, etc.
    • fieldName: is the name of the UI element.
    • xpos: is the position in Horizontal, simply from the prefix x which means horizontal, and pos which stands for position.
    • ypos: is the position in Vertical, simply from the prefix y which means vertical, and pos which stands for position.
    • zpos: is the position in front and back, simply it's like you are editing a video or photo and there are various layers, so the higher the number the more in front of other UI elements, the smaller or minus the more behind other UI elements.
    • wide: is the width of the element.
    • tall: is the height of the element.
    • visible: type value is boolean 0 is hide, 1 is visible on screen.
    • enabled: the type value is boolean 0 is off or not working, 1 is on or working.
    • labeltext: type value string can also be numeric, the example above labeltext is filled with "%HealthNumber%", sign % is to use a variable with a dynamic value HealthNumber file can only be accessed inside the localplayerpanel.res and localplayerpanel_incap.res, This `labeltext` property only exists on controls such as Label, Button, RichText, and related.
    • textAlignment: is the align position of the text, this property only exists on controls such as Label, Button, RichText, and other related, multiple values can be used:
      "west" = left
      "east" = right
      "north" = top
      "south" = bottom
      "north-west" = top left
      "south-west" = bottom left
      "north-east" = top right
      "south-east" = bottom right
    • font: determines what kind of font is used for the element itself, the example above with the name HUDHealth, This font control is in the scheme file, in the case above is the file localplayerpanel.res this file is related to clientscheme.res to set the scheme control in the file, you can find it and edit it, even you can also create a new font control there.
    • fgcolor_override: property to give color to the text.
    • bgcolor_override: property to give color to the background.
    • defaultFgcolor_override: same as above, the property to give the default color to the text, usually I only use it for controls like Label by forcing the element, along with adding this.
    • defaultBgcolor_override: property to give the background a default color.

Custom Scheme Color Then Implement To Specific Resource List

If we previously changed the specific font color of the control UI resource list file, now we will change the font color in one control UI resource list file but take a custom color from the Scheme file such as clientscheme.res, We will create the HealthNumber element again but with a custom color scheme implemented into the UI element, follow the steps:

  • Back in the Text Editor, open localplayerpanel.res then create a new UI element under the `NewHealthNumber` element as below:

    my_mod > resource > ui > hud > localplayerpanel.res

    // Other Code ... "NewHealthNumber" // New HealthNumber with clientscheme.res { "ControlName" "Label" "fieldName" "NewHealthNumber" "xpos" "86" "ypos" "32" "wide" "70" "tall" "26" "visible" "1" "enabled" "1" "labelText" "%HealthNumber%" "textAlignment" "west" "font" "HUDHealth" "zpos" "2" "fgcolor_override" "0 255 255 255" "defaultFgcolor_override" "0 255 255 255" // add this line if above not working } "NewHealthNumber_WithClientScheme" // New HealthNumber with Cyan text color { "ControlName" "Label" "fieldName" "NewHealthNumber_WithClientScheme" "xpos" "116" "ypos" "32" "wide" "70" "tall" "26" "visible" "1" "enabled" "1" "labelText" "%HealthNumber%" "textAlignment" "west" "font" "HUDHealth" "zpos" "2" "fgcolor_override" "MyColorScheme" } }

    As you can see, there on the property fgcolor_override with value MyColorScheme This will look up the color scheme to the file clientscheme.res with the color key MyColorScheme, currently we don't have that color key, let's create it now.
  • Next go to file clientscheme.res we will create a color with key equal to MyColorScheme, name, capitalization, lowercase key color must be the same as the one in the localplayerpanel.res, create above in the Colors scheme, as below:

    my_mod > resource > clientscheme.res

    // // TRACKER SCHEME RESOURCE FILE // // sections: // colors - all the colors used by the scheme // basesettings - contains settings for app to use to draw controls // fonts - list of all the fonts used by app // borders - description of all the borders // // Scheme { //Name - currently overriden in code //{ // "Name" "ClientScheme" //} //////////////////////// COLORS /////////////////////////// Colors { // My Custom Colors "MyColorScheme" "190 0 255 255" // base colors "Orange" "255 176 0 255" "OrangeDim" "255 176 0 120" "LightOrange" "188 112 0 128"

    then save.
  • Next as usual open VPKedit, we compile the "my_mod" folder and save it in the folder:
    "YOUR_STEAM_FOLDER\steamapps\common\Left 4 Dead 2\left4dead2\addons"
  • Open the Left 4 Dead 2 game and see the changes, as below:

    Screenshot:

    And now, we have 3 HealthNumbers, the first is the original, the second HealthNumber is by directly changing the color to the resource file itself, and the third HealthNumber we directly change the color to the resource file itself but with clientscheme.res.

Earlier we have made a simple mod to change the color of a UI element in the font, starting from the first one we have changed the color by changing from the scheme file in the font. clientscheme.res, also we have made new HealthNumber with the name `NewHealthNumber` and assign colors directly to its elements using the property fgcolor_override, and finally we create another HealthNumber but with a property of fgcolor_override with color asign to a file clientscheme.res

And that was a simple way to change the font color of a UI element, try to experiment changing something, add something and see how it changes.
